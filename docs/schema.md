# Схема бази даних бібліотеки

Ця база даних призначена для зберігання інформації про книги, авторів, категорії, читачів та їхні позики.

## Таблиці та опис

### 1. categories
Таблиця містить категорії книг.

| Поле | Тип | Опис |
|------|-----|------|
| id | bigint | Унікальний ідентифікатор категорії |
| name | varchar(100) | Назва категорії |

### 2. books
Таблиця містить інформацію про книги.

| Поле | Тип | Опис |
|------|-----|------|
| id | bigint | Унікальний ідентифікатор книги |
| title | varchar(255) | Назва книги |
| isbn | varchar(20) | ISBN книги |
| version | bigint | Версія книги |
| category_id | bigint | Ідентифікатор категорії (зв'язок з `categories.id`) |

**Зв’язки:**
- `books.category_id` → `categories.id` (один до багатьох: одна категорія може мати багато книг)

### 3. authors
Таблиця містить інформацію про авторів.

| Поле | Тип | Опис |
|------|-----|------|
| id | bigint | Унікальний ідентифікатор автора |
| name | varchar(150) | Ім’я автора |

### 4. book_authors
Таблиця для зберігання зв’язку між книгами та авторами (багато до багатьох).

| Поле | Тип | Опис |
|------|-----|------|
| book_id | bigint | Ідентифікатор книги (зв'язок з `books.id`) |
| author_id | bigint | Ідентифікатор автора (зв'язок з `authors.id`) |

**Зв’язки:**
- `book_authors.book_id` → `books.id`
- `book_authors.author_id` → `authors.id`

### 5. readers
Таблиця містить інформацію про читачів.

| Поле | Тип | Опис |
|------|-----|------|
| id | bigint | Унікальний ідентифікатор читача |
| full_name | varchar(200) | Повне ім’я читача |
| email | varchar(150) | Email читача |
| is_deleted | boolean | Позначка про видалення |
| deleted_at | timestamp | Дата видалення |
| updated_at | timestamp | Дата останнього оновлення |

### 6. loans
Таблиця з інформацією про позики книг читачами.

| Поле | Тип | Опис |
|------|-----|------|
| id | bigint | Унікальний ідентифікатор позики |
| book_id | bigint | Ідентифікатор книги (зв'язок з `books.id`) |
| reader_id | bigint | Ідентифікатор читача (зв'язок з `readers.id`) |
| loan_date | date | Дата позики |
| due_date | date | Дата повернення |
| return_date | date | Фактична дата повернення |

**Зв’язки:**
- `loans.book_id` → `books.id`
- `loans.reader_id` → `readers.id`

## Пояснення схеми

- **Категорії та книги:** кожна книга належить до однієї категорії.
- **Книги та автори:** зв’язок багато до багатьох через таблицю `book_authors`.
- **Читачі та позики:** кожна позика пов’язує конкретного читача з конкретною книгою.
- **Книги та позики:** одна книга може мати багато записів про позики, але одна позика завжди стосується однієї книги та одного читача.

Ця структура дозволяє ефективно зберігати дані про бібліотеку, відстежувати доступність книг та історію позик.
